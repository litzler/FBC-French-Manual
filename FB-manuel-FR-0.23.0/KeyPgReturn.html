Return
 
Déclaration de contrôle de flux pour revenir d'une procédure ou d'un GoSub.

Syntaxe

Return [ expression ]
ou
Return [ label ]


Description

Return est utilisé pour revenir d'une procédure ou d'un GoSub.

Comme Return signifie retour-d'un-gosub ou retour-d'une-procédure, Option Gosub et Option Nogosub peuvent être utilisés pour activer et désactiver le support de GoSub. Quand le support de GoSub est désactivé, Return est alors reconnu comme retour-d'une-procédure. Quand le support de GoSub est activé, Return est alors reconnu comme retour-d'un-gosub.

Return (d'une procédure) est utilisé dans une procédure pour sortir de la procédure peut-être avec une valeur de retour. Un Sub ne peut pas spécifier une valeur de retour. Dans une Function, Return doit préciser sa valeur de retour. Return expression est à peu près équivalent à Function = expression : Exit Function.

Return (d'un gosub) est utilisé pour redonner le contrôle à l'instruction qui suit immédiatement le dernier appel à GoSub. Lorsqu'il est utilisé en combinaison avec GoSub, aucune valeur de retour ne peut être spécifiée. Si l'option label est spécifiée, l'exécution se poursuit à l'étiquette spécifiée. Si aucun GoSub n'avait été exécuté, une erreur d'exécution est générée et l'exécution continue immédiatement après Return.

Un GoSub devrait toujours avoir un Return correspondant. Toutefois, si Return (depuis un gosub) est utilisé lorsqu'aucun GoSub n'a été exécuté, une erreur d'exécution est générée.

Exemple

'' GOSUB & RETURN example, compile with "-lang qb" or use "$lang" as below

'$lang: "qb"

Print "Let's Gosub!"
GoSub MyGosub
Print "Back from Gosub!"
Sleep
End

MyGosub:
Print "In Gosub!"
Return


'' Return from function

Type rational              '' simple rational number type
    numerator As Integer
    denominator As Integer
End Type

'' multiplies two rational types
Function rational_multiply( r1 As rational, r2 As rational ) As rational

    Dim r As rational
    '' multiply the divisors ...
    r.numerator   = r1.numerator   * r2.numerator
    r.denominator = r1.denominator * r2.denominator

    '' ... and return the result
    Return r

End Function

Dim As rational r1 = ( 6, 105 )   '' define some rationals r1 and r2
Dim As rational r2 = ( 70, 4 )
Dim As rational r3

r3 = rational_multiply( r1, r2 )  '' multiply and store the result in r3

'' display the expression
Print r1.numerator & "/" & r1.denominator; " * ";
Print r2.numerator & "/" & r2.denominator; " = ";
Print r3.numerator & "/" & r3.denominator


Différences de dialecte

  • Dans le dialecte -lang fb Return signifie toujours retour-d'une-procédure.
  • Dans le dialecte -lang qb, Return signifie retour-d'un-gosub par défaut sauf si cela a été changé avec Option Nogosub, auquel cas le compilateur reconnaîtra Return comme retour-d'une-procédure.
  • Dans le dialecte -lang fblite, Return signifie retour-d'une-procédure par défaut sauf si cela a été changé avec Option Gosub, auquel cas le compilateur reconnaîtra Return comme retour-d'un-gosub.

Différences avec QB

  • Aucune en utilisant le dialecte -lang qb.
