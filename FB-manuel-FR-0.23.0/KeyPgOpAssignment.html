Opérateur = (Affectation)
 
Affecte une valeur à une variable

Syntaxe

Declare Operator Let ( ByRef lhs As T1, ByRef rhs As T2 )

Usage

lhs = rhs

ou, dans le dialecte QB,

[ Let ] lhs = rhs

Paramètres

lhs
La variable qui reçoit l'affectation.
T1
Une chaîne ou un type pointeur.
rhs
La valeur à assigner à lhs.
T2
Tout type convertibleT2.

Description

Cet opérateur affecte la valeur de son opérande de droite (rhs) à son opérande de gauche (lhs). L'opérande de droite doit être implicitement convertible dans le type de celui de gauche (T1). Par exemple, vous ne pouvez pas assigner une valeur numérique à un type de chaîne; pour le faire, vous devez d'abord convertir la valeur numérique en une chaîne à l'aide de Str ou WStr.

Evitez la confusion avec Opérateur = (Egal), qui utilise également le symbole '='.

Cet opérateur peut être surchargé pour des types définis par l'utilisateur.

Exemple

Dim i As Integer
i = 420    ' <- this is the assignment operator

If  i = 69 Then   '<-this is the equivalence operator 
  Print "ERROR: i should equal 420"
  End -1
End If

Print "All is good."
End 0


' compile with -lang fblite or qb

#lang "fblite"

Dim i As Integer
Let i = 300 ' <-alternate syntax


Différences de dialecte

  • Dans le dialecte -lang qb, cet opérateur ne peut pas être surchargé.
  • Dans le dialecte -lang qb, une expression d'affectation peut être précédée par le mot-clé Let.

Différences avec QB

  • None
