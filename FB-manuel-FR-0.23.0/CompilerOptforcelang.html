Option du compilateur: -forcelang
 
Permet QuickBASIC ou la compatibilité descendante

Syntaxe

-forcelang dialect

Paramètres

dialect
Le dialecte à utiliser pour la compilation, l'un de ceux-ci fb (par défaut), fblite, qb ou deprecated.

Description

L'option du compilateur -forcelang modifie la façon dont le code source est interprété, et est conçu comme un outil pour les utilisateurs souhaitant un comportement semblable à un QuickBASIC traditionnel, ou le comportement obsolète des versions précédentes de FreeBASIC. Elle remplace les déclarations #lang dans le code.

La macro intrinsèque __FB_LANG__ La macro intrinsèque est réglée sur la chaîne nom du dialecte spécifié sur la ligne de commande, ou "fb" par défaut.

Pour en savoir plus sur les différences entre chacun de ces dialectes du langage, voir Les dialectes du compilateur.

fb

C'est le dialecte par défaut, et il permet la compilation du code source en respectant la version la plus récente du langage FreeBasic.

fblite

Ce dialecte fournit un support pour la syntaxe et la fonctionnalité de FreeBasic mais avec un style de programmation QuickBASIC plus traditionnel.

qb

Ce dialecte offre le meilleur support du code QuickBASIC plus ancien.

deprecated

Ce dialecte est pour la compatibilité descendante avec les versions précédentes de FreeBasic, cependant, ce dialecte peut ne plus exister dans les versions futures. Les programmeurs doivent envisager d'utiliser plutôt le dialecte "fblite".
