Field
 
Spécifie l'alignement.

Syntaxe

Type typename Field = { 1 | 2 | 4 }
...
End Type

Description

Note: Les informations suivantes sont susceptibles d'être incorrectes, FB suit les règles de remplissage de GCC, que les phrases suivantes ne décrivent pas. Le fichier source symb-struct.bas du compilateur montre la méthode correcte.

Une variable Type a ses champs alignés sur la longueur d'un mot dans le système, 4 octets présentement. Field modifie l'alignement par défaut à 1 octet (Field 1) ou 2 octets (Field 2). Ceci pour aider FreeBASIC à maintenir la compatibilité avec des structures crées dans les autres langages.

Exemple

Type bitmap_header Field = 1
    bfType          As UShort
    bfsize          As UInteger
    bfReserved1     As UShort
    bfReserved2     As UShort
    bfOffBits       As UInteger
    biSize          As UInteger
    biWidth         As UInteger
    biHeight        As UInteger
    biPlanes        As UShort
    biBitCount      As UShort
    biCompression   As UInteger
    biSizeImage     As UInteger
    biXPelsPerMeter As UInteger
    biYPelsPerMeter As UInteger
    biClrUsed       As UInteger
    biClrImportant  As UInteger
End Type

Dim bmp_header As bitmap_header

'Open up bmp.bmp and get its header data:
'Note: Will not work without a bmp.bmp to load . . .
Open "bmp.bmp" For Binary As #1

    Get #1, , bmp_header
    
Close #1

Print bmp_header.biWidth, bmp_header.biHeight

Sleep



Différences de dialecte

  • Dans le dialecte -lang qb, l'alignement Field est toujours de 1, jamais de remplissage, comme dans QB.

Différences avec QB

  • Dans QB, Field est utilisé pour définir les champs dans un tampon fichier au démarrage. Cette fonctionnalité n'est pas implémentée dans FB, de sorte que le mot-clé a été redéfini. Pour définir des champs dans un tampon de fichier, il faut utiliser Type plusieurs fois.
