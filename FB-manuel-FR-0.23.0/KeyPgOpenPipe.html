Open Pipe
 
Ouvre un processus externe de flux d'entrée (stdin) ou de sortie (stdout) pour des opérations sur les fichier.

Syntaxe

Open Pipe shell_command For Input As [#]filenumber
Open Pipe shell_command For Output As [#]filenumber
Open Pipe shell_command For Binary access_type [#]filenumber

Usage

result = Open Pipe( command, For {Input|Output}, As filenumber )
ou,
result = Open Pipe( command, For Binary, access_type, As filenumber )
(ou dans la syntaxe QB-like,)
Open Pipe filename For {Input|Output} As filenumber
(ou,)
Open Pipe filename For Binary access_type As filenumber

Paramètres

shell_command
Le processus externe à exécuter dans le shell de commande du système d'exploitation. Les chemins relatifs de fichiers sont relatifs au répertoire courant (voir CurDir). Lors de l'ouverture d'un pipe pour un processus qui exige des guillemets doubles dans le chemin de son exécutable ou de ses arguments, la chaîne complète du pipe doit être imbriquée à l'intérieur de guillemets doubles.
access_type
Le type d'accès en lecture ou écriture demandé par le processus appelant.
  • Access {Read|Write} (le flux stdin ou stdout du processus externe sera ouvert)
filenumber
Un numéro de fichier disponible pour se lier au flux stdin ou stdout du processus externe.

Return Value

Dans la première utilisation, Open Pipe retourne zéro (0) si succès et sinon un code d'erreur différent de zéro.

Description

Open Pipe exécute un autre processus dans le shell de commande et ouvre l'un des deux flux stdin ou stdout pour une lecture ou une écriture. Un filenumber (= numéro de fichier) est lié au flux et est utilisé dans les opérations de fichier ultérieures, telles que Input #. Un filenumber (= numéro de fichier) utilisable peut être déterminé avec FreeFile. Si le processus externe n'existe pas, une erreur d'exécution est émise.

Les file modes Input et Output ouvrent, respectivement, les flux stdin et stdout du processus externe, pour une entrée/sortie texte, très utile pour la lecture ou l'écriture du texte brut. Les caractères, les mots ou lignes de caractères peuvent ensuite être lus ou écrits à l'aide des opérations en mode texte sur les fichiers, commme Line Input # et Print #.

Le file mode Binary ouvre les flux stdin ou stdout du processus externe - en fonction de l'access type (= type d'accès) spécifié (voir plus loin la description du paramètre access_type) - pour des accès-aléatoires (random-access) en lecture ou écriture de données brutes de taille arbitraire. Des valeurs de données de types simples, comme Byte et LongInt, et des pans entiers de la mémoire peuvent être lus ou écrits sur les flux avec des opérations de fichier en mode binaire comme Get # et Put #.

Les pipes bidirectionnels ne sont pas supportés par FB et doivent être implémentés en utilisant les fonctions API de l'OS.
Différences de plate-forme

  • Le file mode Binary n'est pas supporté par toutes les plates-formes; Open Pipe produira une erreur s'il ne peut pas ouvrir les flux stdin ou stdout du processus externe en mode binaire.



Erreurs de fonctionnement:
Open Pipe produit l'une des erreurs de fonctionnement suivantes:

(1) Illegal function call : Appel illégal de fonction
  • filenumber n'est pas libre actuellement. Utilisez FreeFile pour vous assurer que filenumber est libre.


Exemple

Dim text As String
Open Pipe "fbc.exe" For Input As #1
Print "Output of fbc:"
Do While Not EOF(1)
   Line Input #1, text
   Print text
Loop
Close #1
