Inkey
 
Renvoie une chaîne représentant la première touche en attente dans la mémoire tampon du clavier

Syntaxe

Declare Function Inkey ( ) As String

Usage

result = Inkey[$]

Valeur retournée

Le premier caractère trouvé dans le tampon de clavier ou une chaîne vide ("") si rien n'est trouvé.

Description

Vérifie rapidement dans la mémoire tampon du clavier et retourne une représentation String du premier caractère, s'il y en a un de trouvé. La touche est ensuite retirée de la mémoire tampon et n'est pas envoyée à l'écran. Si la mémoire tampon du clavier est vide, une chaîne vide ("") est retournée immédiatement, sans attendre d'autres touches.

Si la valeur est dans le jeu de caractères ASCII, une String d'un seul caractère correspondant à cette valeur est retournée. Si la touche est une touche "étendue" (pavé numérique, curseur, fonctions) une String de deux caractères est retournée, dans laquelle le premier représente le caractère étendu (Voir ci-dessous différences de dialecte)

Les touches Shift, Ctrl, Alt et AltGr ne peut pas être lues indépendamment par cette fonction car elles n'entrent jamais dans la mémoire tampon du clavier (bien que, de toute évidence, Shift-A sera signalé par Inkey différemment de Control-A et ainsi de suite; Alt-A est une touche étendue comme ci-dessus).

Voir aussi Input() ou GetKey, ou Sleep pour attendre une touche si la mémoire tampon du clavier est vide.

Exemple

Print "press q to quit"
Do
    Sleep 1, 1
Loop Until Inkey = "q"

#if __FB_LANG__ = "qb"
#define EXTCHAR Chr$(0)
#else
#define EXTCHAR Chr(255)
#endif

Dim k As String

Print "Press a key, or Escape to end"
Do

    k = Inkey$

    Select Case k

        Case "A" To "Z", "a" To "z": Print "Letter: " & k
        Case "1" To "9":             Print "Number: " & k

        Case Chr$(32): Print "Space"

        Case Chr$(27): Print "Escape"

        Case Chr$(32) To Chr$(127)
            Print "Printable character: " & k

        Case EXTCHAR & "G": Print "Up Left / Home"
        Case EXTCHAR & "H": Print "Up"
        Case EXTCHAR & "I": Print "Up Right / PgUp"

        Case EXTCHAR & "K": Print "Left"
        Case EXTCHAR & "L": Print "Center"
        Case EXTCHAR & "M": Print "Right"

        Case EXTCHAR & "O": Print "Down Left / End"
        Case EXTCHAR & "P": Print "Down"
        Case EXTCHAR & "Q": Print "Down Right / PgDn"

        Case EXTCHAR & Chr$(59) To EXTCHAR & Chr$(68)
            Print "Function key: F" & Asc(k, 2) - 58

        Case EXTCHAR & Chr$(133) To EXTCHAR & Chr$(134)
            Print "Function key: F" & Asc(k, 2) - 122

        Case Else
            If Len(k) = 2 Then
                Print Using "Extended character: chr$(###, ###)"; Asc(k, 1); Asc(k, 2)
            ElseIf Len(k) = 1 Then
                Print Using "Character chr$(###)"; Asc(k)
            End If

    End Select

    If k = Chr$(27) Then Exit Do

    Sleep 1, 1

Loop


Différences de dialectes

  • Le caractère étendu est Chr(255) dans les dialectes -lang fb et -lang fblite.
    • Dans le dialecte -lang qb, le caractère étendu dépend de la façon dont le mot-clé est écrit. Si la forme Inkey$ est utilisée, le caractère étendu est Chr(0). Si elle est référencée comme __Inkey, le char est étendu Chr(255).
    • Dans tous les autres dialectes, le caractère étendu est toujours Chr(255).
  • Le suffixe de type chaîne $ est facultatif dans les dialectes -lang fblite et -lang fb, mais nécessaire en dialecte -lang qb.

Différences avec QB

  • Aucune dans le dialecte -lang qb.
  • QBasic retourne un Chr(0) comme premier caractère pour une touche étendue, mais FreeBASIC retourne Chr(255) comme premier caractère dans les dialectes -lang fb et -lang fblite.
