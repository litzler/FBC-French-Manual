Asc
 
Retourne l'entier correspondant à la représentation ASCII ou Unicode d'un caractère

Syntaxe

Declare Function Asc Overload ( ByRef str As Const String, ByVal position As Integer = 1 ) As UInteger
Declare Function Asc ( ByVal str As Const ZString Ptr, ByVal position As Integer = 1 ) As UInteger
Declare Function Asc ( ByVal str As Const WString Ptr, ByVal position As Integer = 1 ) As UInteger

Usage

result = Asc( str [, position ] )

Paramètres

str
La chaîne source.
position
La position d'un caractère dans une chaîne.

Valeur retournée

L'entier correspondant à la représentation ASCII ou Unicode d'un caractère situé à position dans str.

Description

Si str est une String ou une ZString, un code ASCII est retourné.

Si str est une WString, un code Unicode est retourné.

La fonction retourne zéro (0) si la chaîne a une longueur nulle, si position est inférieure à (1), ou si position est supérieure au nombre de caractères dans str.

Chr exécute l'opération inverse pour les chaînes de caractères ASCII, alors que WChr est l'opposé pour les chaînes Unicode, retournant une chaîne de caractères contenant le caractère représenté par le code passé en argument.

Exemple

Print "the ascii code of 'a' is:"; Asc("a")
Print "the ascii code of 'b' is:"; Asc("abc", 2)

produira la sortie:

the ascii code of 'a' is: 97
the ascii code of 'b' is: 98

ExempleUnicode (Note aux rédacteurs de documentation: ne mettez pas dans %%(qbasic) marqueurs ou le texte russe va disparaître!)

dim a as wstring * 11
a = "Привет, мир"
print "the Unicode of the second char of " & a & " is: " & asc(a)
 
produira la sortie:
the Unicode of the second char of Привет, мир is: 1088
 

Différences de plate-forme

  • DOS ne supporte pas la version chaîne avec larges caractères de Asc.

Différences avec QB

  • L'argument optionnel position est nouveau pour FreeBASIC.
  • QB ne supporte pas la version chaîne avec larges caractères de Asc
