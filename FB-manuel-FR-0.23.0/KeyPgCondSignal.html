CondSignal
 
Redémarre un "thread" suspendu par un appel à CondWait

Syntaxe

Declare Sub CondSignal ( ByVal handle As Any Ptr )

Usage

CondSignal ( handle )

Paramètres

handle
The handle of a conditional variable, or the null pointer (0) on failure.

Description

Une fois que le conditionnel est CondCreate et les "threads" démarrés, l'un (ou plusieurs) d'entre eux peut être fixé à CondWait pour le conditionnel, ils seront arrêtés jusqu'à un autre "thread" CondSignal que le "thread" en attente puisse redémarrer. CondBroadcast peut être utilisé pour redémarrer tous les "threads" qui attendent le conditionnel. A la fin du programme CondDestroy doit être utilisé pour éviter les fuites de ressources dans le système d'exploitation.

Exemple

Voir CondCreate

Différences de dialecte

  • "Threading" n'est pas utilisable dans le dialecte -lang qb

Différences de plate-forme

  • Condsignal n'est pas utilisable avec la version/cible DOS de FreeBASIC, parce que le "multithreading" n'est pas supporté par le noyau DOS, ni l'extension utilisée.
  • Sous Linux les "threads" sont toujours démarrés dans l'ordre de leur création, ceci n'est pas vrai sous Win32. C'est un OS, pas une question FreeBasic.

Différences avec QB

  • Nouveau pour FreeBASIC
