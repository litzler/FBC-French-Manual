Utilisation de bibliothèques pré-construites
 
FreeBasic est distribué avec de nombreuses en-têtes pour des bibliothèques communes ou populaires. Les en-têtes permettent à un programmeur d'utiliser les fonctions disponibles dans ces bibliothèques existantes statiques ou partagées (DLL).

Les bibliothèques elles-mêmes ne sont pas distribuées avec FreeBasic, mais la plupart peuvent être téléchargées à partir du web et installées facilement. D'autres bibliothèques peuvent avoir besoin, en premier, d'être compilées à partir de sources avant d'être utilisées. S'il vous plaît, consultez la documentation des bibliothèques spécifiques sur la façon de les configurer, de les installer et les utiliser.

Certaines bibliothèques statiques ou partagées (DLL) peuvent être déjà présentes sur le système car elles peuvent faire partie de FreeBasic lui-même ou du système d'exploitation.

Bien que de nombreuses en-têtes peuvent être utilisées sur n'importe quelle plate-forme supportée par FreeBasic, certaines sont spécifiques à certaines plate-formes et non-utilisables sur d'autres.

En-têtes FreeBASIC

Il y a quelques en-têtes qui sont spécifiques à FreeBasic et proposent certaines fonctions qui, autrement, ne sont pas disponibles:
  • datetime.bi - Déclarations pour DateSerial, DateValue, IsDate, Year, Month, Day, Weekday, TimeSerial, TimeValue, Hour, Minute, Second, Now, DateAdd, DatePart, DateDiff, MonthName, WeekdayName
  • dir.bi - Constantes à utiliser avec Dir
  • fbgfx.bi - Constantes et structures supplémentaires à utiliser avec les commandes graphiques telles que MultiKey, ScreenControl, et ScreenEvent, ImageCreate.
  • file.bi - Déclarations pour FileCopy, FileAttr, FileLen, FileExists, FileDateTime
  • string.bi - Déclarations pour Format
  • vbcompat.bi - Inclut datetime.bi, dir.bi, file.bi, et string.bi plus des constantes supplémentaires compatibles avec Microsoft Visual Basic.

C Runtime (CRT)

Lorsque cela est possible des en-têtes compatibles multiplateformes ont été fournies pour C run-time (CRT). Par exemple,
#include once "crt.bi"
printf( !"Hello World\n" )

Pour inclure un fichier d'en-tête CRT spécifique, préfixer le nom du fichier d'en-tête avec "crt/". Par exemple:
#include once "crt/stdio.bi"
Dim f As FILE Ptr
f = fopen("somefile.txt", "w")
fprintf( f, "Hello File\n")
fclose( f )


Windows API

Beaucoup (de nombreuses) en-têtes pour les API Windows sont disponibles pour inclusion dans le code source FreeBasic. Dans la plupart des cas le seul fichier nécessaire à inclure est "windows.bi". Par exemple,
#include once "windows.bi"
MessageBox( null, "Hello World", "FreeBASIC", MB_OK )

Pour inclure une l'en-tête spécifique de l'API Windows, préfixer nom de l'en-tête avec "win/" par exemple:
#include once "win/ddraw.bi"

Examinez le répertoire "./inc/win/" où FreeBASIC a été installé pour voir tous les en-têtes disponibles pour les API Windows.

Autres en-têtes fournis

Examinez le répertoire "./inc/" où FreeBASIC a été installé pour voir tous les autres en-têtes disponibles. Il est possible que des en-têtes soient disponibles pour une bibliothèque, vous devez utiliser. Certaines en-têtes sont situées dans "./inc/" et les autres peuvent se trouver un sous-répertoire. Pour inclure les en-têtes situées dans un sous-répertoire "./inc/", préfixez le nom de l'en-tête avec le nom du répertoire où il se trouve. Par exemple:
'' located in ./inc/
#include once "curl.bi"

'' located in ./inc/GL
#include once "GL/gl.bi"


Exigences pour l'utilisation de bibliothèques statiques prédéfinies

  • Le code source doit inclure les en-têtes appropriées à l'aide de #include.
  • Une bibliothèque statique doit être liée à la compilation en utilisant #inclib dans le code source ou en utilisant l'option -l sur ligne de commande pour spécifier le nom de la bibliothèque.

Exigences pour l'utilisation de bibliothèques partagées prédéfinies

  • Le code source doit inclure les en-têtes appropriées à l'aide de #include.
  • La bibliothèque partagée (.DLL) doit être présente sur l'ordinateur hôte sur lequel programme compilé s'exécute.
