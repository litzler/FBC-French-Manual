Les définitions intrinsèques
 
Symboles de préprocesseur définis par le compilateur.

Description

Les définitions intrinsèques sont fixées par le compilateur et peuvent être utilisées comme n'importe quel autre symbole défini. Les définitions intrinsèques souvent transmettre des informations sur l'état du compilateur, soit en général, soit à un moment précis dans le processus de compilation. La plupart des définitions intrinsèques sont associées à une valeur.

Information sur la plate-forme
Définitions qui fournissent des informations sur le système.
Information sur la version
Définitions qui fournissent des informations sur la version du compilateur fbc en usage.
Commutateurs de ligne de commande
Définitions qui fournissent des informations sur les commutateurs de ligne de commande utilisés avec fbc.
Information sur l'environnement
Définitions qui fournissent des informations sur l'environnement du système d'exploitation.
Information sur le contexte spécifique
Définitions qui fournissent des informations sur le contexte spécifique du processus de compilation.

Information sur la plate-forme
__FB_WIN32__
Défini si compilation pour Windows.
__FB_LINUX__
Défini si compilation pour Linux.
__FB_DOS__
Défini si compilation pour DOS.
__FB_CYGWIN__
Défini si compilation pour Cygwin.
__FB_FREEBSD__
Défini si compilation pour FreeBSD.
__FB_NETBSD__
Défini si compilation pour NetBSD.
__FB_OPENBSD__
Défini si compilation pour OpenBSD.
__FB_DARWIN__
Défini si compilation pour Darwin.
__FB_XBOX__
Défini si compilation pour Xbox.
__FB_BIGENDIAN__
Défini si compilation pour un système utilisant un ordre des octets "big-endian".
__Fb_Pcos__
Défini si compilation pour un OS PC commun (e.g. DOS, Windows, OS/2).
__Fb_Unix__
Défini si compilation pour un OS Unix.

Information sur la version
__FB_VERSION__
Défini comme une chaîne littérale de la version du compilateur.
__FB_VER_MAJOR__
Défini comme le numéro de version majeure du compilateur.
__FB_VER_MINOR__
Défini comme le numéro de version mineure du compilateur.
__FB_VER_PATCH__
Défini comme le numéro de révision de la version du compilateur.
__FB_MIN_VERSION__
Macro pour tester la version minimum du compilateur.
__FB_BUILD_DATE__
Défini comme la date de compilation du compilateur.
__FB_SIGNATURE__
Défini comme la signature du compilateur.

Commutateurs de ligne de commande
__FB_Backend__
Défini si "gas" ou "gcc" dépendent de -gen.
__Fb_Gcc__
Vrai (True) (-1) si -gen gcc est utilisé, sinon faux (false) (0).
__FB_MAIN__
Défini si compilation d'un module avec un point d'entrée.
__FB_DEBUG__
Vrai (-1) si l'option "-g" est utilisée, sinon faux (0).
__FB_ERR__
Zéro (0) si aucun des commutateurs "-e", "-ex" ou "-exx" n'est utilisé.
__Fb_Fpmode__
Défini comme "fast" si compilation pour les mathématiques SSE rapides, sinon "precise".
__Fb_Fpu__
Défini comme "sse" si compilation unité SSE à virgule flottante, ou "x87" pour une unité x87 normale à virgule flottante.
__FB_LANG__
Défini comme une chaîne littérale du dialecte "-lang" utilisé.
__FB_MT__
Vrai (-1) si le commutateur "-mt" est utilisé, sinon faux (0).
__FB_OUT_DLL__
Vrai (-1) dans un module qui est compilé et lié dans une bibliothèque partagée, sinon faux (0).
__FB_OUT_EXE__
Vrai (-1) dans un module qui est compilé et lié dans un fichier exécutable, sinon faux (0).
__FB_OUT_LIB__
Vrai (-1) dans un module qui est compilé et lié dans une bibliothèque statique, sinon zéro (0).
__FB_OUT_OBJ__
Vrai (-1) dans un module qui est seulement compilé, sinon zéro (0).
__FB_SSE__
Défini si compilation pour unité SSE à virgule flottante.
__Fb_Vectorize__
Défini comme niveau de vectorisation automatique (0 à 2)
Information sur l'environnement
__FB_ARGC__
Défini comme un littéral entier représentant le nombre d'arguments passés au programme par la ligne de commande .
__FB_ARGV__
Défini comme un ZString Ptr Ptr vers les arguments de la ligne de commande passés au programme.
__DATE__
Défini comme une chaîne littérale de la date de compilation.
__TIME__
Défini comme une chaîne littérale de l'heure de la compilation.
__PATH__
Défini comme une chaîne littérale du chemin absolu du module.

Information sur le contexte spécifique
__FILE__ et __FILE_NQ__
Défini comme le nom du module.
__FUNCTION__ et __FUNCTION_NQ__
Défini comme le nom de la procédure où il est utilisé.
__LINE__
Défini comme un littéral entier de la ligne du module où il est utilisé.
__FB_OPTION_BYVAL__
Vrai (-1) si les paramètres sont déclarés par la valeur, par défaut, sinon zéro (0).
__FB_OPTION_DYNAMIC__
Vrai (-1) si tous les tableaux sont de longueur variable, sinon zéro (0).
__FB_OPTION_ESCAPE__
Vrai (-1) si les chaînes littérales sont traitées avec des séquences d'échappement, sinon zéro (0).
__Fb_Option_Gosub__
Vrai (-1) si le support de gosub est activé, sinon zéro (0).
__FB_OPTION_EXPLICIT__
Vrai (-1) si les variables et les objets doivent être explicitement déclarés, sinon zéro (0).
__FB_OPTION_PRIVATE__
Vrai (-1) si toutes les procédures sont privées (private) par défaut, sinon zéro (0).
