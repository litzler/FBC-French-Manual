Préprocesseur
 
Les commandes qui contrôlent le préprocesseur.

Description

Les commandes du préprocesseur sont envoyées au compilateur pour contrôler ce qui est compilé et comment. Elles peuvent être utilisées pour choisir de compiler un bloc de code plutôt qu'une autre pour la compatibilité multi-plateforme, les en-têtes ou autres fichiers source, définir de petites fonctions "inline" (en ligne) appelées macros ou de modifier la façon dont le compilateur gère les variables.

Compilation conditionnelle
Commandes qui permettent des branchements dans la compilation en fonction de conditions.
Texte de remplacement
Commandes qui créent des macros de texte de remplacement.
Directives fichiers
Commandes qui indiquent au compilateur comment lier d'autres fichiers au fichier source.
Directives de contrôle
Commandes qui définissent des options de compilation, le contrôle de la compilation et rapporte des informations sur le déroulement de la compilation.
Métacommandes
Les commandes qui sont conservées pour la compatibilité descendante.

Compilation conditionnelle
#if
Compilation du bloc de code qui suit, basée sur une condition.
#ifdef
Compilation du bloc de code qui suit, si un symbole est défini.
#ifndef
Compilation du bloc de code qui suit, si un symbole n'est pas défini.
#elseif
Compilation du bloc de code qui suit, si une condition est vraie et que les conditions précédentes étaient fausses.
#else
Compilation du bloc de code qui suit, si les conditions précédentes étaient fausses.
#endif
Signifie la fin d'un bloc de code.
defined
Retourne "-1" si un symbole est défini, sinon "0".

Texte de remplacement
#define
Crée une macro d'une seule ligne de texte de remplacement.
#macro et #endmacro
Crée une macro multi-ligne de texte de remplacement.
#undef
Annule la définition d'un symbole.
# Preprocessor Stringize
Convertit le texte dans une chaîne littérale.
## Preprocessor Concatenate
Concaténation de deux morceaux de texte.
! Escaped String Literal
Indique que le littéral de chaîne immédiatement après doit être traité avec des séquences d'échappement.
$ Non-Escaped String Literal
Indique que le littéral de chaîne immédiatement après ne doit pas être traité avec des séquences d'échappement.

Directives fichiers
#include
Insère du texte à partir d'un fichier.
#inclib
Intègre une bibliothèque dans le processus de liaison.
#libpath
Intègre un chemin de recherche pour les bibliothèques dans le processus de liaison.

Directives de contrôle
#pragma
Définit des options de compilation.
#lang
Définit le dialecte de la source.
#print
Affiche un message sur la sortie standard lors de la compilation.
#error
Affiche un message sur la sortie standard et arrête la compilation.
#line
Définit le numéro de ligne courant et nom de fichier.

Métacommandes
'$Include
Autre forme de la directive #include.
'$Dynamic
Autre forme de l'instruction Option Dynamic.
'$Static
Autre forme de l'instruction Option Static.
'$Lang
Autre forme de la directive #lang.
