Fonctions de gestion du "threading"
 
Procédures pour travailler avec des applications "multi-thread".

Description

Ces procédures permettent la programmation "multi-thread". "Threads" et variables conditionnelles peuvent être créés et détruits, et les mutexes peuvent être obtenus pour protéger les données de "thread" sensibles..

Threads
Les procédures de départ et d'attente pour les procédures de "thread".
Varables conditionnelles
Procédures qui créent et signalent des variables conditionnelles.
Mutexes
Méthodes qui traîtent avec les "mutex".

Threads
ThreadCreate
Démarre une procédure dans un "thread" séparé de l'exécution.
ThreadWait
Attend jusqu'Ã  ce que une procédure de "thread" soit terminée.

Conditional Variables
CondCreate
Crée une variable conditionnelle.
CondWait
Suspend l'exécution d'une procédure de "thread".
CondSignal
Reprend l'exécution d'une procédure de "thread" en attente d'une valeur conditionnelle.
CondBroadcast
Reprend toutes les procéduresde "thread" en attente d'une valeur conditionnelle.
CondDestroy
Détruit une variable conditionnelle qui n'est plus nécessaire.

Mutexes
MutexCreate
Crée un "mutex".
MutexLock
Acquiert un verrou sur un "mutex".
MutexUnlock
Annule un verrou sur un "mutex".
MutexDestroy
Détruit un "mutex" qui n'est plus nécessaire.
 
Différences de plate-forme

  • Ces procédures ne sont pas supportées par le DOS.
