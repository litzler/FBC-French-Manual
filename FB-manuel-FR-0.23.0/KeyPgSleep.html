Sleep
 
Attend jusqu'à ce qu'un certain délai soit écoulé ouqu'une touche soit appuyée.

Syntaxe

Declare Function Sleep Overload ( ByVal amount As Integer = -1 ) As Integer
Declare Function Sleep ( ByVal amount As Integer , ByVal keyflag As Integer ) As Integer

Usage

Sleep [ amount [, keyflag ]]

Paramètres

amount
Nombre facultatif de millisecondes à attendre (par défaut on attend qu'une touche soit appuyée).
keyflag
Drapeau optionnel; lui donner une valeur de 1 sert à préciser que l'attente ne peut être interrompue par une touche.

Description

Sleep attendra jusqu'à ce que amount millisecondes (peut être en secondes dans -lang qb, voir plus loin) soit écoulé (si une valeur a été passée) ou jusqu'à ce que l'utilisateur appuie sur une touche. Si amount est inférieur à 100 ms, alors Sleep attendra toujours le temps demandé au complet (appui sur touche ignoré).

Inclure le second paramètre, 1, pour un «profond» du sommeil, qui ne peut être interrompu en appuyant sur une touche.

La précision de Sleep est variable en fonction de la durée du cycle du système d'exploitation (Windows NT/2K/XP: 15 ms, 9x/Me: 50 ms, Linux 10ms, DOS 55 ms).

Appeler Sleep avec 25ms ou moins pour réaliser un découpage du temps lors de l'attente d'une entrée utilisateur ou d'une boucle dans un "thread". Cela permettra d'éviter que le programme monopolise inutilement le CPU.

Sleep n'efface pas la mémoire tampon du clavier et les touches pressées lors d'un appel à Sleep sont conservée et peuvent être lues avec Inkey. Afin d'attendre l'appui sur une touche et retirer la clé du tampon mémoire, GetKey peut être utilisé à la place.

Exemple

Print "press a key"
Sleep
GetKey 'clear the keyboard buffer
Print "waiting half second"
Sleep 500


Différences de dialecte

  • Dans les dialectes -lang fb et -lang fblite, la valeur de amount est en millisecondes.
  • Dans le dialecte -lang qb, la valeur de amount est en secondes comme dans QB. Si le deuxième paramètre keyflag est donné, ou le mot-clé __Sleep est donné, la valeur est attendue en millisecondes.

Différences avec QB

  • Aucune dans le dialecte -lang qb.
  • Dans QB, le délai est donné secondes et le paramètre keyflag n'est pas supporté.
