ByVal
 
Spécificateur de déclaration pour passer explicitement un paramètre par valeur

Syntaxe

ByVal param As datatype

Usage

[ Declare ] { Sub | Function } proc_name ( ByVal param As datatype )

Description

ByVal dans une liste de paramètres d'une instruction de déclaration provoque une copie de la variable à passer par sa valeur à la procédure (par exemple, un sous-programme ou une fonction).

Cela signifie que si la valeur de la variable x est passée, alors la variable d'origine x ne sera en aucune façon modifiée; toutefois, si la variable a été passée ByRef, la valeur de la variable d'origine x peut être modifiée par la fonction appelée.

Note: Pour arguments de chaîne, ByVal a actuellement un sens différent: au lieu de passer une copie de la chaîne, il passe un pointeur ZString vers la chaîne de données, alors que ByRef passe un pointeur vers le descripteur de chaîne FreeBASIC. Ce comportement permet de passer ByVal String directement aux procédures C. La chaîne passée de cette manière ne sera pas changée par Sub/Function, et le descripteur de chaîne ne sera pas mis à jour. La signification du passage d'une String ByVal pourrait changer dans le futur; par conséquent, passer String ByVal doit être évité. ZString est plus adapté dans le but de passer des "chaînes terminées par zéro" (ZString) aux fonctions C.

L'opposé de ByRef.

Exemple

Sub MySub(ByVal value As Integer)
    value += 1
End Sub

Dim MyVar As Integer

MyVar = 1
Print "MyVar: "; MyVar 'output = 1
MySub MyVar
Print "MyVar: "; MyVar 'output = 1, because byval won't change the values passed into it globally.
Sleep
End


Différences de dialecte

  • Dans le dialecte -lang fb, ByVal est la convention de passage de paramètre par défaut pour tous les types intégrés à l'exception de String; Chaîne (String) et les Types définis par l'utilisateur sont passés ByRef par défaut.
  • Dans les dialectes -lang qb et -lang fblite, ByRef est la convention de passage des paramètres.

Différences avec QB

  • QB utilise seulement ByVal dans les déclarations aux sous-routines non-Basic
