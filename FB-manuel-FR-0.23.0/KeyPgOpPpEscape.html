Opérateur ! (Chaîne littérale avec séquences d'échappement)
 
Indique explicitement qu'une chaîne littérale supporte les séquences d'échappement.

Syntaxe

!"text"

Paramètres

!
L'opérateur préprocesseur de support des séquences d'échappement
"text"
La chaîne littérale contenant des caractères d'échappement

Description

Cet opérateur indique explicitement que la chaîne littérale qui le suit (entre guillemets doubles) peut être traîtée pour le support des séquences d'échappement. C'est un opérateur du préprocesseur et il ne peut être utilisé qu'avec des chaînes littérales lors de la compilation.

Le comportement par défaut pour les chaînes littérales c'est qu'elles ne seront pas traitées comme supportant les séquences d'échappement. Option Escape peut être utilisée dans le dialecte -lang fblite pour annuler ce comportement par défaut, entraînant alors le support des séquences d'échappement pour toutes les chaînes.

Utilisez l'opérateur $ Operator (Non Echappement chaîne littérale) pour indiquer explicitement qu'une chaîne ne supporte pas le traîtement des séquences d'échappement.

Exemple

Print "Some escape sequence examples:"
Print !"1.\tsingle quote (\\\') : \'"
Print !"2.\tdouble quote (\\\") : \""
Print !"3.\tbackslash    (\\\\) : \\"
Print !"4.\tascii char   (\\65): \65"

'' OUTPUT:
''
'' Some escape sequence examples:
'' 1. single quote (\') : '
'' 2. double quote (\") : "
'' 3. backslash    (\\) : \
'' 4. ascii char   (\65): A
