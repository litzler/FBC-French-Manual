WString
 
Type de données standard : chaînes de caractères larges (2 octets)

Syntaxe

Dim variable As WString * size
Dim variable As WString Ptr

Description

Une WString est un tableau à taille fixe de caractères larges qui ne déborde jamais si la taille est connue au moment de la compilation. Il n'y a pas de descripteur, et ne peut Ãªtre redimensionné sauf si c'est un pointeur et que Allocate/Reallocate/Deallocate sont utilisés directement.

La fin de la chaîne est marquée avec le caractère 0, donc ce caractère ne doit jamais faire partie du contenu de WString ou ce contenu sera tronqué.

Ce type existe pour supporter les alphabets avec une base non-Latin. Toutes les fonctions intrinsèques de chaînes comme Left , ainsi que tout opérateur de chaîne, fontionnent aussi avec WString.

En plus des fichiers ASCII avec des séquences d'échappement Unicode (\u), FreeBasic peut analyser les fichiers source UTF-8, UTF-16LE, UTF-16BE, UTF-32LE et UTF-32BE.

Les fonctions de fichier texte de FreeBasic peuvent lire et écrire des fichiers Unicode dans des codages différents, à condition que le codage = Encoding soit spécifié lorsque le fichier est ouvert. Le texte est automatiquement converti dans le codage interne à la lecture et reconverti dans le codage du fichier lors de l'écriture.

SizeOf( WString ) retourne le nombre d'octets utilisés par WString dans la plate-forme courante.

Exemple

Dim As WString * 14 str1 => "hello, world"
Print str1
Print Len(str1)    'returns 12, the length of the string it contains 
Print SizeOf(str1) 'returns 14 * sizeof(wstring), the number of bytes used by the variable


Dim As WString Ptr str2
str2 = Allocate( 14 * Len(WString) )
*str2 = "hello, world"
Print *str2
Print Len(*str2)      'returns 12, the length of the string it points to


Différences de plate-forme

Support for wstrings relies in the C runtime library available in the platform and the internal format may vary.
  • Unicode n'est pas supporté dans le portage DOS de FreeBASIC. Dans ce portage un caractère vaut toujours 1 octet et Wstrings va se comporter en tant que norme ASCII Zstrings
  • Sous Win32 wstrings sont encodées en UCS-2 (UTF-16 LE) et un caractère vaut 2 octets.
  • Sous Linux wstrings sont encodées en UCS-4 et un caractère vaut 4 octets.

Différences de dialectes

  • Non disponible dans le dialecte -lang qb sauf si référencé avec l'alias __Wstring.

Différences avec QB

  • Nouveau pour FreeBASIC
