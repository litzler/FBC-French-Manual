Err
 
Fonction de gestion des erreurs qui renvoie le numéro de la dernière erreur

Syntaxe

Declare Function Err ( ) As Integer

Usage

result = Err

Return Value

Après une erreur, retourne le code de l'erreur qui s'est produite.

Description

Err peut toujours être utilisé, même si la gestion des erreurs QB-like n'est pas activée.

Err est réinitialisé par Resume et Resume Next.

NOTE: Des précautions doivent être prises lors de l'appel d'une fonction interne (comme Print) après qu'une erreur se soit produite car la valeur d'erreur sera réinitialisée avec son propre statut d'erreur. Pour préserver la valeur Err, il faut la copier dans une variable auxiliaire dès que le gestionnaire d'erreurs est lancé.

Voir Codes d'erreur de fonctionnement pour une liste des numéros d'erreur d'exécution et leur signification.

Exemple

Un exemple en utilisant le style QBasic de gestion d'erreur (compilation avec l'option -ex)
'' Compile with -lang fblite or qb

#lang "fblite"

On Error Goto Error_Handler
Error 150
End

Error_Handler:
  n = Err()
  Print "Error #"; n
  Resume Next

Un exemple en utilisant la gestion des erreurs en ligne (note: Open peut aussi retourner son propre status d'erreur s'il est appelé comme une fonction)
'' compile without -e switch

Dim filename As String

Do
    Line Input "Input filename: ", filename
    If filename = "" Then End
    Open filename For Input As #1
Loop Until Err() = 0

Print Using "File '&' opened successfully"; filename
Close #1


Différences avec QB

  • Les numéros d'erreur sont différents dans QB.
