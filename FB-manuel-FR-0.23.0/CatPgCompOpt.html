Options du compilateur
 
Les options de commande en ligne du compilateur pour le compilateur fbc:

@< file >
    • Lire options de ligne de commande (supplémentaires) à partir du fichier
-a < name >
    • Ajouter un fichier objet à la liste éditeur de liens
-arch < type >
    • Définir l'architecture cible (défaut: 486)
-b < name >
    • Ajouter un fichier source à la compilation
-c
    • Compiler seulement, ne pas lier
-C
    • Ne pas supprimer le(s) fichier(s) objet
-d < name=val >
    • Ajouter un "define" préprocesseur
-dll
    • Créer une DLL, y compris la bibliothèque d'importation. (Identique à -dylib)
-dylib
    • Créer une DLL, y compris la bibliothèque d'importation
-e
    • Ajouter la vérification des erreurs
-ex
    • Ajouter la vérification des erreurs avec le support de RESUME
-exx
    • Identique à -ex plus vérification des limites de tableaux et de pointeurs NULL
-export
    • Exporter les symboles pour la liaison dynamique
-forcelang <name>
    • Compatibilité de sélection de langage, annule #lang/$lang dans le code
-fpmode < type >
    • Choix entre rapide et précise pour les opérations à virgule flottante (défaut: PRECISE)
-fpu < type >
    • Régle l'unité arithmétique virgule flottante (défaut: FPU)
-g
    • Ajouter les informations de débogage
-gen < backend >
    • Définit le compilateur "backend" (défaut est 'gas')
-i < name >
    • Ajouter un chemin de recherche pour les fichiers include
-include < name >
    • Inclure un fichier en-tête pour chaque source compilé
-l < name >
    • Ajouter un fichier de bibliothèque à la liste éditeur de liens
-lang < name >
    • Sélectionne compatibilité de langage: fb, fblite, qb, deprecated
-lib
    • Créer une bibliothèque statique
-m < name >
    • Fichier principal sans extension, le point d'entrée (par défaut est le premier fichier. bas sur la ligne de commande)
-map < name >
    • Enregistrer la carte de l'édition des liens sous un nom de fichier
-maxerr < val >
    • Stoppe l'analyse d'erreurs seulement si survient l'erreur <val>
-mt
    • Lien avec bibliothèque "thread-safe runtime"
-nodeflibs
    • Ne pas inclure les bibliothèques par défaut
-noerrline
    • Ne pas afficher la ligne du source où l'erreur s'est produite
-o < name >
    • Définir le chemin/nom du fichier objet (doit être passé après le fichier. bas)
-O < level >
    • Définir le niveau d'optimisation (-gen gcc)
-p < name >
    • Ajouter un chemin de recherche pour les bibliothèques
-pp
    • Produire seulement le fichier d'entrée prétraîté, ne pas compiler
-prefix < path >
    • Définir le préfixe du chemin du compilateur
-profile
    • Activer la fonction de profilage
-r
    • Ecrire seulement des fichiers asm, ne pas compiler ou lier
-R
    • Ne pas supprimer le(s) fichier(s) asm
-s < name >
    • Définir le sous-système (gui, console)
-target < platform >
    • Définir la plate-forme cible pour la compilation croisée
-t < value >
    • Définir la taille de la pile en kilo-octets (défaut: 1M)
-v
    • Mode "bavard"
-vec < level >
    • Indiquer le niveau des optimisations permises par le compilateur (défaut: 0)
-version
    • Affiche la version du compilateur
-w < value >
    • Indiquer le niveau d'alerte min: all, pedantic ou une valeur
-Wa < opt >
    • Passe les options à GAS (séparées par des virgules)
-Wc < opt >
    • Passe les options à GCC (séparées par des virgules)
-Wl < opt >
    • Passe les options à LD (séparées par des virgules)
-x < name >
    • Définir exécutable/bibliothèque chemin/nom
-z < value >
    • Définit des options diverses ou expérimentales
