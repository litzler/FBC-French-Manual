Règles d'identification
 
Conventions de nommage pour les symboles FreeBasic.

Description

Un identificateur est un nom symbolique qui identifie de manière unique une variable, un Type, une Union, une Enum, une Function, une Sub ou Property, dans sa portée ou son espace de noms.

Les identificateurs ne peuvent contenir que des lettres majuscules et minuscules en caractères Latin (a-z et A-Z), les chiffres (0-9) et le caractère de soulignement (_). Le premier caractère d'un identificateur doit être une lettre ou un trait de soulignement, pas un chiffre.

Les identifiants ne sont pas sensibles à la casse: FOO et foo (et toutes les autres permutations de lettres majuscules et minuscules) se réfèrent à un même symbole.

Dans les dialectes -lang qb et -lang fblite, les identificateurs peuvent avoir un suffixe de type indiquant l'un des types de données standard:

  • % for Integer
  • & for Long
  • ! for Single
  • # for Double
  • $ for String

L'utilisation de ces symboles est généralement déconseillée et n'est pas autorisée dans le dialecte -lang fb (par défaut).

L'alternative est d'être explicite - par exemple, Dim As Integer foo ou Dim foo As Integer au lieu de Dim foo%.

Dans les dialectes -lang qb et -lang fblite, les identificateurs peuvent contenir un ou plusieurs points (.).

Différences de dialecte

  • Les points dans les noms des symboles sont pris en charge uniquement dans les dialectes -lang qb et -lang fblite.

Différences avec QB

  • Le support du caractère de soulignement (_) dans les noms de symboles est nouveau pour FreeBasic.
