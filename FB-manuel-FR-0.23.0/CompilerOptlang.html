Option du compilateur: -lang
 
Fournit la compatibilité descendante ou QuickBASIC

Syntaxe

-lang dialect

Paramètres

dialect
Le dialecte à utiliser lors de la compilation, un des suivants fb (défaut), fblite, qb ou deprecated.

Description

L'option du compilateur -lang L'option du compilateur modifie la manière dont le code source est interprété et se veut un outil pour les utilisateurs qui souhaitent un comportement traditionnel de type QuickBASIC ou un comportement obsolète des versions précédentes de FreeBASIC.

La macro intrinsèque __FB_LANG__ est définie sur le nom de chaîne du dialecte spécifié sur la ligne de commande ou "fb" par défaut.

Pour en savoir plus sur les différences entre chacun de ces dialectes du langage, voir Les dialectes du compilateur.

fb

C'est le dialecte par défaut qui permet la compilation d'un code source adhérant à la version la plus récente du langage FreeBASIC.

fblite

Ce dialecte fournit un support pour la syntaxe et la fonctionnalité FreeBasic, mais avec une programmation de style QuickBASIC plus traditionnel.

qb

Ce dialecte offre le meilleur soutien pour l'ancien code de QuickBASIC.

deprecated

Ce dialecte existe pour la compatibilité descendante avec les versions précédentes de FreeBasic, cependant, ce dialecte pourrait disparaître dans les versions futures. Les programmeurs doivent envisager d'utiliser plutôt le dialecte "fblite".

Note: cette option de ligne de commande peut être annulée par les déclarations de #lang utilisées dans le code.
