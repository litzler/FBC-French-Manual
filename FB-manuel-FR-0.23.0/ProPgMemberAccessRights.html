Vue d'ensemble

Tous les membres d'un Type ou d'une Class - inclus les données des membres, les procédures, les constantes, etc. - appartiennent à l'une de trois catégories différentes, chacune avec ses propres règles dictant où dans le code ils pourront être consultés ou référencés. Ces règles sont appelées des droits d'accès. Il y a des membres publics, protégés et privés, et ils sont déclarés dans la définition d'un Type ou d'une Class suivant, respectivement, un label Public, Protected ou Private.

Par défaut, sans label de classification d'accès, les membres d'un Type sont publics, et les membres d'une Class sont privés.

Membres publics (Public)

On peut faire référence à des membres publics de n'importe quel endroit; ils sont accessibles, par exemple, à partir de membres de procédures, ou de code ou procédures au niveau module.

Membres protégés (Protected)

Les membres protégés ne sont accessibles qu'à partir de membres de procédures Type ou Class où ils ont été déclarés, ou des membres de procédures d'un Type ou d'une Class dérivé(e). Ils ne sont pas accessibles à l'extérieur du code.

Membres privés (Private)

Les membres privés ne sont accessibles qu'à partir de membre de procédures Type ou Class où ils ont été déclarés. Ils ne sont pas accessibles à l'extérieur du code ou de membre de procédures de Type ou Class dérivé(e).

Constructeurs et destructeurs

Constructeurs et destructeurs suivent les mêmes règles que tout autre membre. Lorsqu'ils sont publics, les objets peuvent être instanciés et détruits de partout dans le code. Lorsqu'ils sont protégés, les objets ne peuvent être instanciés et détruits qu'à partir des membres de procédures de leur Type ou Class ou Type ou Class dérivé(e). Les constructeurs et destructeurs privés restreignent l'instanciation d'objets aux seuls membres de procédures de leur Type ou Class.
